# 🎮 Codequest — คู่มือทำต่อพรุ่งนี้ (มีคอม)

ทุกอย่างพร้อมแล้ว วันนี้แค่เก็บไฟล์นี้ไว้ พรุ่งนี้เปิดคอมแล้วทำตามได้เลย ใช้เวลารวม **30-45 นาที**

---

## ✅ สิ่งที่มีอยู่แล้วตอนนี้

โฟลเดอร์ `pixel-rpg-learn/` มีเว็บที่ **ใช้งานได้จริง** ทันที ไม่ต้องติดตั้งอะไรเลย:

- ✅ หน้า Dashboard (ตัวละคร, XP bar, Coin, Streak)
- ✅ ระบบ Quest (ทำเควสต์ → ได้ XP/Coin → เลเวลอัป)
- ✅ ระบบ Story (ปลดล็อกตามเควสต์)
- ✅ Rival AI (เทียบเวลาเรียน)
- ✅ บันทึกความคืบหน้าอัตโนมัติ (localStorage — ปิดเว็บแล้วเปิดใหม่ ข้อมูลไม่หาย)
- ⚠️ ตัวละครตอนนี้เป็น **sprite ชั่วคราว** (ผมวาดแบบง่ายๆ ให้ดูได้ก่อน) — ขั้นที่ 2 จะเปลี่ยนเป็นของจริง

---

## 📋 ขั้นที่ 1: เปิดเว็บดูก่อน (2 นาที)

1. แตกไฟล์ `pixel-rpg-learn.zip` ที่ส่งให้
2. ดับเบิลคลิกไฟล์ `index.html` → เปิดด้วย Chrome/Edge/Firefox
3. คุณจะเห็นหน้าเกมทันที ลองกด "เริ่มเควสต์" ดู

ถ้าเปิดแล้วเห็นหน้าเกม (มีตัวละคร, แถบ XP, เควสต์) แปลว่า**สำเร็จขั้นแรก** 🎉

---

## 🎨 ขั้นที่ 2: เปลี่ยนเป็น Pixel Sprite จริง (15 นาที)

### 2.1 ดาวน์โหลด sprite ฟรี

ไปที่ลิงก์เหล่านี้ (ฟรีทั้งหมด ไม่ต้องสมัครสมาชิก):

| ใช้สำหรับ | ลิงก์ | คำแนะนำ |
|---|---|---|
| ตัวละครผู้เล่น (Wizard) | https://craftpix.net/freebies/free-wizard-sprite-sheets-pixel-art/ | กด "Free Download" |
| ตัวละครสำรอง (Knight) | https://craftpix.net/freebies/ → ค้นหา "knight free" | ถ้าอยากเปลี่ยนเป็นนักรบแทน |
| Rival ghost | https://craftpix.net/freebies/ → ค้นหา "ghost monster free" | |
| Health/XP Bar UI | https://itch.io/game-assets/free/tag-health-bar | เลือก "Basic Pixel Health bar and Scroll bar" โดย BDragon1727 |
| ไอคอน (เหรียญ, ดาบ, ม้วนกระดาษ) | https://itch.io/game-assets/free/tag-gui/tag-pixel-art | เลือก "8000+ Raven Fantasy Icons" — มีครบทุกอย่างที่ต้องใช้ |

### 2.2 แตกไฟล์ที่ดาวน์โหลดมา

แต่ละไฟล์จะเป็น `.zip` → แตกไฟล์ออกมา จะเจอไฟล์ `.png` หลายไฟล์ (แต่ละท่าทาง/แอนิเมชัน)

### 2.3 เลือกรูปแล้ววางทับ

หาไฟล์ที่หน้าตาเป็น **"idle" หรือ "stand"** (ท่ายืนเฉยๆ) ของ Wizard แล้ว:

1. เปลี่ยนชื่อไฟล์เป็น `idle.png`
2. ก็อปปี้ไปวางที่ `pixel-rpg-learn/assets/sprites/wizard/idle.png` (ทับไฟล์เดิม)

ทำแบบเดียวกันกับ:
- Ghost/monster → `assets/sprites/icons/rival.png`
- ไอคอนเหรียญ → `assets/sprites/icons/coin.png`
- ไอคอนไฟ/streak → `assets/sprites/icons/streak.png`
- ไอคอนดาบ → `assets/sprites/icons/sword.png`
- ไอคอนม้วนกระดาษ → `assets/sprites/icons/scroll.png`
- ไอคอนดาว → `assets/sprites/icons/star.png`

**เคล็ดลับ**: ไม่ต้องรู้โค้ดเลย แค่ลากไฟล์ไปวางทับ ชื่อไฟล์ต้องตรงเป๊ะ (ตัวเล็กตัวใหญ่มีผล)

### 2.4 รีเฟรชเว็บ

กด `Ctrl+R` หรือ `F5` ที่หน้าเว็บ — ตัวละครจะเปลี่ยนเป็น pixel art จริงทันที ไม่ต้องแก้โค้ดอะไรเลย!

## 🌐 ขั้นที่ 3: เอาขึ้นเว็บจริง เปิดจากมือถือได้ทุกที่ (15 นาที)

ตอนนี้เว็บอยู่ในคอมคุณคนเดียว ขั้นนี้จะทำให้เปิดจากมือถือที่ไหนก็ได้ผ่านลิงก์

### 3.1 สมัคร GitHub (ถ้ายังไม่มี)
→ https://github.com/signup (ฟรี)

### 3.2 สร้าง repository ใหม่
1. กดปุ่ม "+" มุมขวาบน → "New repository"
2. ตั้งชื่อ `codequest`
3. เลือก Public
4. กด "Create repository"

### 3.3 อัปโหลดไฟล์
1. ในหน้า repo ใหม่ กด "uploading an existing file"
2. ลากทั้งโฟลเดอร์ `pixel-rpg-learn` (หรือไฟล์ข้างในทั้งหมด) ไปวาง
3. กด "Commit changes"

### 3.4 สมัคร Vercel แล้วเชื่อมกับ GitHub
1. ไปที่ https://vercel.com/signup
2. เลือก "Continue with GitHub"
3. กด "Add New Project"
4. เลือก repo `codequest` ที่เพิ่งสร้าง
5. กด "Deploy" (ไม่ต้องตั้งค่าอะไรเพิ่ม เพราะเป็น HTML ธรรมดา)
6. รอ 30 วินาที → ได้ลิงก์ เช่น `codequest-yourname.vercel.app`

### 3.5 เปิดจากมือถือ
เอาลิงก์ที่ได้ไปเปิดในมือถือ (Safari/Chrome) → กด "Add to Home Screen" เพื่อให้เหมือนเป็นแอป

🎉 **เสร็จแล้ว!** ตอนนี้คุณมีเว็บเกมเรียนโค้ดของตัวเอง เปิดจากมือถือได้ทุกที่ และทุกครั้งที่แก้ไฟล์ใน GitHub เว็บจะอัปเดตอัตโนมัติใน 30 วินาที (ตาม Pipeline ที่คุณระบุไว้ใน spec เป๊ะเลย)

## 🤖 ขั้นต่อไป (หลังจากนี้): ต่อ AI Agent

ตอนนี้ระบบ "ตรวจคำตอบ" ยังเป็นแบบง่ายๆ (รับทุกคำตอบที่ไม่ว่าง) เพราะยังไม่ได้ต่อ AI จริง

ขั้นต่อไปที่ผมจะช่วยสร้างให้คือ:
1. **Validator Agent** — ส่งคำตอบไปให้ AI (Claude/GPT/DeepSeek) ตรวจจริง ให้ feedback
2. **Teacher Agent** — สร้างเควสต์ใหม่อัตโนมัติตามจุดอ่อนของคุณ
3. **Multi-provider settings** — หน้า UI ให้เลือกว่าแต่ละ Agent ใช้ AI ตัวไหน

อันนี้ต้องใช้ **Backend/API** (ซับซ้อนกว่าเว็บ static ตอนนี้) — เดี๋ยวค่อยคุยรายละเอียดกันอีกทีตอนขั้นนี้เสร็จ

## 🆘 ติดปัญหา?

|---|---|
| เปิด index.html แล้วเป็นหน้าขาว | เช็คว่าโฟลเดอร์ครบ (css/, js/, assets/) ไม่ได้แยกไฟล์ออกจากกัน |
